# Product Requirements Document

# 07 - Notifications

---

# Overview

The Notification System is designed to encourage users, improve consistency, and strengthen communication between users and mentors.

Notifications should always feel supportive.

The application must never use language that creates guilt or discouragement.

---

# Objectives

The notification system should:

- Encourage daily tracking.
- Increase consistency.
- Celebrate achievements.
- Remind users gently.
- Help mentors follow up.
- Improve engagement.

---

# Notification Channels

Version 1 supports:

- In-App Notifications
- Email Notifications

Future versions may support:

- Push Notifications
- WhatsApp Integration
- SMS

---

# Notification Types

The system supports several notification categories.

- Daily Reminder
- Weekly Reminder
- Achievement
- Badge
- Promotion
- Weekly Review
- Mission
- Mentor Reminder
- System Notification

---

# Daily Reminder

Purpose

Remind users to complete today's worship.

Example Messages

"Today is another opportunity to continue your journey."

"Keep your consistency alive today."

"You are doing great. Record today's progress."

Daily reminders should be configurable.

---

# Missed Day Reminder

If the user does not submit today's tracking.

Example Messages

"You can always start again."

"A small step today is better than waiting for tomorrow."

The message should encourage action.

Never shame the user.

---

# Weekly Reminder

At the end of every week.

Notify the user that:

Weekly Review is available.

Mentor feedback has arrived.

The week has been completed.

---

# Achievement Notification

Triggered automatically.

Examples

"You unlocked your first badge."

"You completed your first week."

"Great job! You reached 100 Quran pages."

---

# Promotion Notification

When a mentor promotes the user.

Example

"Congratulations!

Your mentor promoted you to Level 3."

---

# Mission Notification

Notify users when:

- New mission assigned.
- Mission completed.
- Mission deadline approaching.

---

# Mentor Notifications

Mentors receive notifications when:

- A user stops recording activities.
- Weekly Review is due.
- A user reaches promotion criteria.
- A mission is completed.

---

# Region Admin Notifications

Receive notifications when:

- New mentor created.
- Mentor inactive.
- Weekly reports generated.
- Region statistics available.

---

# Super Admin Notifications

Receive notifications for:

- Critical system errors.
- New Region.
- New Administrator.
- System Reports.
- Security Events.

---

# Notification Preferences

Every user can configure:

- Email On/Off
- In-App On/Off

Future versions may include:

- Notification Schedule
- Quiet Hours

---

# Read Status

Notifications have:

Unread

Read

Archived

Users may mark notifications as read.

---

# Notification History

Every notification should be stored.

History includes:

- Title
- Message
- Type
- Created At
- Read At
- Sender (if applicable)

---

# Delivery Rules

Duplicate notifications should not be sent.

Notifications should be delivered only once.

Failed notifications should be retried automatically.

---

# Language

Notifications should be sent in the user's selected language.

Supported:

Arabic

English

---

# Acceptance Criteria

✓ Daily reminders supported.

✓ Weekly reminders supported.

✓ Achievement notifications supported.

✓ Mission notifications supported.

✓ Promotion notifications supported.

✓ Read status supported.

✓ Notification history stored.

✓ Multi-language supported.

✓ Positive language only.
