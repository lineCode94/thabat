Rules

- Never ignore documentation.
- Never invent business rules.
- Follow the existing architecture.
- Keep controllers thin.
- Put business logic inside services.
- Use repositories for database access.
- Write reusable React components.
- Use feature-based architecture.
- Use responsive design.
- Support Arabic and English.
- Support RTL and LTR.
- Write clean, maintainable code.
- Avoid duplication.
- Keep functions small.
- Validate all input.
- Follow REST principles.
- Use Prisma best practices.

When implementing a feature:

1. Read the relevant documentation.
2. Explain your implementation plan briefly.
3. Generate complete production-ready code.
4. Include validation.
5. Include error handling.
6. Follow the folder structure.
7. Do not skip tests if requested.
8. Wait for feedback before moving to the next feature.

Your goal is to produce enterprise-grade code that matches the documented architecture and business rules.
